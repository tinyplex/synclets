createFileConnectors

The createFileConnectors function creates both a file-backed DataConnector and MetaConnector together in a single call, returning them as a tuple.

createFileConnectors<Depth>(
  depth: Depth,
  file: string,
  options?: FileConnectorsOptions,
): Connectors<Depth, FileDataConnector<Depth>, FileMetaConnector<Depth>>
TypeDescription
depthDepth

The tree depth the Synclet will operate at.

filestring

The base file path (used for both connectors if options not provided).

options?FileConnectorsOptions

Optional configuration to use separate file paths.

returnsConnectors<Depth, FileDataConnector<Depth>, FileMetaConnector<Depth>>

A tuple of [DataConnector, MetaConnector].

This is the recommended approach for file-based persistence in Node.js. The function provides a simplified API compared to creating the connectors separately.

When options are not provided, both data and metadata are stored in the same file. For better organization and to avoid potential conflicts in large datasets, you can specify separate file paths via the options parameter.

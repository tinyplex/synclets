Context

The Context type holds arbitrary key-value metadata that accompanies sync messages and mutation operations. Context values must be Atoms (strings, numbers, booleans, null, or UNDEFINED). Common uses include authentication tokens, user identifiers, request IDs, or any other information needed for authorization, routing, or auditing.

{[key: string]: Atom}
